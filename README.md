# Flix カメラシミュレータ

プログラミング言語 [Flix](https://flix.dev/)（JVM で動く関数型言語）で書いた、
ゲームの「追従カメラ」のシミュレータです。星空に浮かぶアリーナで四角（box）を
動かしながら、2D ゲームのカメラの基本要素を一通り体験・観察できます。

- **追従** … カメラが box を少し遅れて追いかける
- **境界** … ステージの端まで来ると、カメラだけがそこで止まる
- **ズーム** … ＋/− ボタンで滑らかに寄り引きする
- **視差** … 遠くの星ほどゆっくり流れて、奥行きがあるように見える
- **画面固定 UI** … ＋/− ボタンは、カメラが動いても画面に貼り付いたまま
- **PiP ミニビューポート** … 右下の小窓にプレイヤー周辺の本物の絵を縮小表示（clip のデモ）

![カメラショーケース: 端まで歩くとリングの向こうに星空が見えてカメラが止まり、ズームアウトするとアリーナが星空に浮かぶ全景になる](docs/showcase.gif)

見どころ: 右の壁まで歩く → カメラが星空の縁で止まる → 縁に沿って歩くと星と床の
流れる速さがずれる（これが視差）→ ズームアウトでアリーナの全景 → 寄り直してループ。

（この GIF は手で録画したものではなく、`make bake` が操作の台本から自動生成しています。
ゲームの計算が乱数や実時間に頼らないので、同じ台本からいつでも同じ映像を作り直せます。）

## PiP ミニビューポート（エンジンの clip のデモ）

画面右下の小窓（PiP）に、プレイヤーを中心とした world 480×320 を 0.2 倍で縮小表示します。
専用のミニマップ絵は持たず、メイン画面と同じ部品（床タイル・アリーナの縁・プレイヤー）を
そのまま映すので、本物と地図が食い違いません。中身は必ず窓から溢れるため、エンジンの
clip 機能（`Render.clipped` / `Render.clippedAll`）で窓の縁ぴったりに切り落としています。
アリーナの端に立つと、縮小タイルが窓の途中でスパッと途切れて虚空に変わるのが見どころです。

![PiP ミニビューポート: 右下の小窓にプレイヤー周辺の縮小ビュー。アリーナ左端でタイルが窓の中で途切れる](golden/pip.png)

（このスクリーンショットは `flix run --entrypoint Shot.all` が画面なしで焼いた

## 前提

必要なのは [devbox](https://www.jetify.com/devbox) を1つ入れることだけです。
JDK21・Flix コンパイラ・make は devbox が（内部で nix を使って）自動でそろえます。
自分で nix や JDK を入れる必要はありません。

## 始め方

```sh
git clone https://github.com/ababup1192/flix_ge_camera.git
cd flix_ge_camera
devbox run make run
```

初回は依存パッケージのダウンロードとコンパイルで少し時間がかかります。窓が開いたら成功です。

### 操作

- **矢印キー**（または **WASD**）… box を動かす
- **右上の ＋/− ボタンをクリック** … ズームイン / ズームアウト

まずは端まで歩いてカメラが止まるところと、− を押し切ってアリーナの全景になるところを
試してみてください。

よく使うコマンド:

| コマンド | 何をするか |
|---|---|
| `devbox run make run`   | シミュレータを起動する（窓が開く） |
| `devbox run make check` | 型検査だけ走らせる（一番速い確認） |
| `devbox run make build` | 実行できる形にビルドする |
| `devbox run make test`  | テストを実行する |

## 触る場所

コードは `src/` にあります。役割はこの4つです。

- `src/World.flix` … シミュレータの状態そのもの（box とカメラがいまどこにいるか）と、次の状態を作る規則。
- `src/Controls.flix` … キーとマウスの割り当て（どの操作で何をするか）。
- `src/View.flix` … 状態を絵に写す（何をどこに描くか。星空の背景もここ）。
- `src/Main.flix` … 上の3つをエンジンに繋いで起動する、目次のようなファイル。

まずは `src/World.flix` の数字を変えて `devbox run make run` し直してみてください。

- `follow()` の `k` … カメラが追いつく速さ。大きいほどキビキビ、小さいほどゆったり
- `zoomMin()` / `zoomMax()` … ズームの範囲。どこまで寄れて、どこまで引けるか

カメラの「気持ちよさ」がどこから来るのか、感覚がつかめます。

`project.json` は窓の大きさ・背景色・使うフォントを決める設定ファイルです。

## 自分のゲームに流用する

カメラまわりの実装（`World.flix` のカメラ状態と追従・境界・ズームの規則、
`View.flix` の星空の視差レイヤ）は、そのまま自分のゲームに切り出して使えます。

## 注記

エンジン本体（[flix_game_engine](https://github.com/ababup1192/flix_game_engine)）は、
初回ビルド時に GitHub Release から自動でダウンロードされます。
ネットワークに繋がらない環境では初回の依存取得に失敗します。
